# 01. Product & Game Design – Dots and Boxes Game System

## 1. Vision & Strategy
- 목표: 고전 Dots and Boxes를 디지털로 재구현하여, 초등 수준의 **전략 사고·공간 지능·수학적 추론**을 놀이를 통해 학습하게 한다.
- 대상: 만 4세 이상 어린이, 초등 1–4학년, 교사·학부모.
- 핵심 방향:
  - 규칙은 단순하지만 전략 깊이는 충분히 제공.
  - 연령·실력별 난이도와 모드를 설계해 "놀이 → 학습 → 전략"을 단계적으로 경험.
  - 교실/가정 양쪽에서 사용 가능한 교육 도구로 포지셔닝.

## 2. 제품 개요 (PRD 요약)
- 제품명(가칭): Dots and Boxes Game System.
- 플랫폼: 웹/모바일(브라우저, 하이브리드 앱).
- 주요 이해관계자:
  - 학생(유아, 초등 저/고학년), 교사, 학부모, 운영/개발 팀.
- 핵심 KPI:
  - 튜토리얼 완료율, 7/30일 리텐션, 교실 모드 사용량, 평균 세션 길이.

## 3. 주요 기능 (High-level)
1) Dots and Boxes 대국(싱글/로컬 2인/온라인 옵션).
2) AI 봇 난이도 선택(쉬움/보통/어려움).
3) 튜토리얼/학습 모드 (규칙 + 기초 전략 학습).
4) 교실 모드 (교사용 대시보드, 학생별 진행 상황).
5) 기본 통계/리포트 (학생·교사·학부모용 요약).
6) 설정/접근성/언어 지원.

## 4. 게임 디자인 (GDD 요약)

### 4.1 핵심 게임 루프
1) 내 차례가 온다.
2) 두 점을 선택해 선을 긋는다.
3) 상자를 완성하면 점수 + 추가 턴.
4) 선을 모두 그리면 게임 종료, 상자 수 비교로 승패 결정.

### 4.2 규칙·승리 조건
- 정사각 격자 위 인접 점 사이에만 선을 그을 수 있다.
- 네 변이 모두 채워진 사각형 상자는 해당 플레이어의 것이다.
- 게임 종료 시 상자 수가 많은 플레이어 승리(동점은 무승부).

### 4.3 변형 규칙(교육용)
- 추가 턴 없음 모드(유아용).
- 제한 시간 모드.
- 퍼즐 모드(특정 상황에서 최선의 수 찾기).

## 5. 모드·난이도 설계

### 5.1 모드별 개요
- 튜토리얼 모드: 규칙/전략을 단계적으로 가르치는 스크립트형 모드.
- 싱글플레이 vs AI: 난이도별 봇과 대국.
- 로컬 2인: 한 디바이스에서 번갈아 플레이.
- 온라인 매칭(후속/옵션): 캐주얼/랭크.
- 교실 모드: 교사 주도 수업용.

### 5.2 난이도 & 연령대 매핑
- 만 4–6세
  - 보드: 3x3~4x4, 추가 턴 없음, 게임 길이 5분 이내.
  - 목표: 상자 만들기 경험, 선 긋기·색칠의 재미.
- 초등 1–2학년
  - 보드: 4x4~5x5, 기본 규칙 적용.
  - 목표: 규칙 이해, 세 번째 변 피하기, 짧은 체인 인식.
- 초등 3–4학년
  - 보드: 5x5~6x6, 체인/루프/이중 교차 전략 도입.
  - 목표: 체인 관리, 상대의 실수를 유도하는 전략.

### 5.3 AI 난이도 개략
- 쉬움: 랜덤성이 크고, 일부 명백한 실수만 회피.
- 보통: 세 번째 변을 되도록 피하고, 짧은 체인 활용.
- 어려움: 체인/루프 구조 분석, 긴 체인은 상대에게 열게 유도, 이중 교차.

## 6. 교육 목표 설계

### 6.1 학습 목표
- 규칙 이해: 상자·변·점 개념, 턴·추가 턴 규칙.
- 전략 이해: 세 번째 변, 체인, 루프, 긴 체인 규칙.
- 메타인지: 자신의 실수 패턴 인식(어디서 상자를 내줬는지).

### 6.2 교실 연계
- 수학 교과(평면도형, 둘레, 패턴)와의 매핑.
- 수업 활동 예시: 리플레이를 보며 다른 선택 토론, 전략 발표.

## 7. 주요 UX 플로우 개요

### 7.1 학생 플로우
1) 앱 실행 → 메인 메뉴.
2) "빠르게 시작" 또는 난이도/모드 선택.
3) (처음이면) 튜토리얼 안내 → 1판 진행.
4) 게임 결과 화면에서 점수·체인 요약·간단 코멘트 확인.
5) 재도전 또는 난이도/모드 변경.

### 7.2 교사 플로우
1) 교사 로그인.
2) 새 교실 세션 생성(학년/보드 크기/목표 선택).
3) 세션 코드 공유, 학생 입장.
4) 대시보드에서 학생 진행 상황 모니터링.
5) 수업 종료 후 요약 확인.

### 7.3 학부모 플로우
1) 자녀 디바이스에서 학부모 뷰 진입.
2) 최근 플레이 기록/난이도/튜토리얼 상태 확인.
3) 추천 활동 및 가정용 게임 제안 확인.
